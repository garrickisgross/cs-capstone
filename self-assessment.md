# Professional Self-Assessment

Completing the Computer Science program and assembling this ePortfolio helped me see my growth more clearly than any single assignment did on its own. Across the program, I developed a stronger foundation in software design, algorithms, databases, and the broader communication skills required to present technical work professionally. Building this portfolio pushed me to do more than finish code that works. It required me to explain decisions, connect technical work to course outcomes, and present my development in a way that would make sense to instructors, peers, and potential employers.

One of the biggest ways this program helped me grow was in learning how to approach software as a system instead of just a collection of files and features. Early on, I focused mostly on making code function. As I progressed, I became more aware of maintainability, separation of concerns, validation, testing, and long-term flexibility. That shift is reflected clearly in this capstone artifact. What began as a simple FastAPI application with routes and in-memory data became a more structured system with routers, services, schemas, optimization logic, and persistent storage.

The program also improved my ability to collaborate and communicate with different audiences. Even when assignments were completed individually, I still had to think about how to explain technical ideas clearly and justify my decisions. In professional settings, I have also seen the value of communicating with stakeholders who care more about outcomes than implementation details. That perspective shaped how I built this portfolio. I wanted the technical work to be strong, but I also wanted the presentation to clearly explain why each enhancement mattered. Writing these narratives reinforced that technical ability alone is not enough. Being able to communicate the purpose, trade-offs, and results of a solution is just as important.

My work with data structures and algorithms became more meaningful when I started applying them to practical problems instead of treating them as isolated exercises. In this artifact, I implemented geographic distance calculation using the Haversine formula, built a distance matrix, and applied a 2-opt optimization algorithm to improve delivery routing. This helped me demonstrate that algorithmic thinking is not just about solving abstract problems. It is also about modeling real-world data, structuring computation effectively, and making deliberate trade-offs based on the needs of the project.

The software engineering and database portions of the program also shaped how I think about application development. In the software design enhancement, I refactored the project from a monolithic structure into a layered application with routers, services, schemas, and a storage abstraction. That work improved maintainability and created a stronger foundation for future changes. In the database enhancement, I replaced temporary in-memory storage with SQLite persistence. That step demonstrated practical database integration within an existing application and showed how abstraction can make architectural changes much easier to implement.

Security is another area where my thinking matured during the program. My capstone is not a fully production-ready system, but the project still reflects a stronger security mindset than my earlier work. I used structured validation with Pydantic, avoided directly coupling user input to persistence logic, and used parameterized SQL operations in the SQLite layer. Just as important, I became more aware of what was still missing, including authentication, stronger access control, and deeper protections around external integrations. That awareness is part of growth too. Security is not only about the protections already implemented, but also about recognizing the risks and planning systems with those risks in mind.

This portfolio brings together three connected views of the same artifact. The software design enhancement shows how I improved the architecture of the application. The algorithms and data structures enhancement shows how I added computational value through route optimization. The database enhancement shows how I made the application more practical and durable by persisting data across sessions. Together, these pages tell a more complete story than any single enhancement could alone. They demonstrate that I can structure software, implement useful algorithms, and integrate persistence in a way that supports a real workflow.

Overall, this capstone helped me see how the pieces of the computer science program fit together. It also helped me present myself more confidently as someone who can design, build, improve, and explain software solutions. This portfolio reflects not only what I created, but how I learned to think more intentionally as a developer.
